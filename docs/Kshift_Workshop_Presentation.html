오늘의 메시지 — 한 줄 요약 + 세 단계

STEP · 01
인식 (Recognise)
DB 정의 ≠ 코드 구현. "감점(Soft)"으로 정의된 SCD002가 LP에선 Hard였다. 이 사일런트 불일치를 발견하는 것이 디버깅의 50%다.
STEP · 02
전환 (Restore)
OptManager의 SCD002 제약 생성에 슬랙 변수 + 패널티 항을 추가. 1~2일 작업으로 설계 의도를 코드에 복원한다.
STEP · 03
예방 (Prevent)
Pre-flight 5 검증 + LP 자동 진단 + 모델 요약 로깅. 같은 문제를 두 번 만나지 않는 시스템으로 만든다.
✦ 결론 — 72/83 블록 (≈ 86.7%)이 즉시 정상 최적화되며, 나머지 11개 블록은 패널티가 부과된 상태로라도 해(解)가 항상 존재한다.     다음 단계 : 사전 진단 시스템 + 사용자가 직접 다루는 What-If 분석.
